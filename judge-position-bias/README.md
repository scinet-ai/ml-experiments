# Order-flip position bias vs. model scale — Qwen2.5-Instruct ladder on MT-Bench pairs

Does the position bias of a pairwise LLM judge decrease with model scale, everything else
pinned? Single-family open-weights ladder (Qwen2.5-Instruct **0.5B / 1.5B / 3B / 7B**),
one pinned protocol, every pair judged in **both presentation orders**.

SciNet problem: `45ee7f2c` · investigation: `cf4e6c02` · agent: `trackc-judge-01`.

## Protocol (pinned)

- **Pairs:** 350 of the 910 unique turn-1 (question, model_a, model_b) pairs of
  [lmsys/mt_bench_human_judgments](https://huggingface.co/datasets/lmsys/mt_bench_human_judgments)
  (split `human`, CC-BY-4.0), after filtering to chat-templated prompts ≤ 3072 tokens;
  deterministic stratified sample by human-majority label, seed 0
  (120 m1 / 119 m2 / 71 tie / 40 no-majority — see `pairs_meta.json`).
- **Judge prompt:** Zheng-et-al-(2306.05685)-style pairwise template (verbatim in
  `judge_eval.py::JUDGE_TEMPLATE`, sha256 prefix `975db383e05f56b3`). It explicitly
  instructs the judge to avoid position and length biases. Verdict options:
  `A` (first better), `B` (second better), `C` (tie).
- **Verdict extraction (deterministic):** one prefill forward pass; argmax over the
  next-token logprobs of the three single-token candidates `A`/`B`/`C` at the first
  assistant position. bf16, Apple-MPS. No sampling anywhere.
- **Both orders:** each pair is judged as (ans1, ans2) and (ans2, ans1).
- **Metrics:**
  - **flip rate** — fraction of pairs where the swapped verdict is not the mirror of the
    original (order changes the outcome);
  - **primacy share** — among flipped pairs, fraction picking slot-1 in *both* orders;
  - **P(pick slot-1)** — overall probability of choosing the first-presented answer;
  - **human agreement** — on order-consistent, non-tie verdicts vs. the human majority label;
  - 95% bootstrap CIs (10k reps, seed 0); pairwise scale deltas with paired bootstrap.
- **Robustness check:** free-generation verdicts (greedy, 8 new tokens, parse A/B/C) on a
  60-pair subsample per model, compared to the logit-based verdicts (`genchk_*.csv`).

## Results

| judge | flip rate [95% CI] | P(pick slot-1) [95% CI] | primacy/recency share of flips | consistent-verdict yield | human agree (n) |
|---|---|---|---|---|---|
| 0.5B | 0.969 [0.949, 0.986] | 0.933 [0.911, 0.953] | 0.91 / 0.00 | 0.031 | 0.800 (5) |
| 1.5B | 0.446 [0.394, 0.497] | 0.700 [0.673, 0.727] | 0.95 / 0.05 | 0.554 | 0.846 (149) |
| 3B | 0.483 [0.429, 0.534] | 0.220 [0.190, 0.250] | 0.08 / 0.41 | 0.517 | 0.817 (82) |
| 7B | 0.189 [0.149, 0.229] | 0.504 [0.481, 0.527] | 0.55 / 0.21 | 0.811 | 0.876 (210) |

Free-generation vs logit verdict agreement: 1.000 at every scale (480/480 parsed).
Adjacent-scale flip deltas (paired bootstrap): 0.5B→1.5B −0.523 [−0.580,−0.469]; 1.5B→3B +0.037 [−0.037,+0.111] (n.s.); 3B→7B −0.294 [−0.360,−0.229].

(`results.json` is the machine-readable source; regenerated by `analyze`.)

## Reproduce

**Zero-download smoke check** (no model, no dataset; regenerates every headline number
from the committed raw verdicts and verifies against the committed `results.json`):

```bash
./reproduce.sh
```

**Full from-scratch run** (~20 GB of model downloads, ~1–3 h on an M4 Max):

```bash
uv venv .venv && uv pip install --python .venv/bin/python -r requirements.txt
.venv/bin/python judge_eval.py prep          # build pair set from the HF dataset
./run_ladder.sh                              # judge + genchk, all 4 model sizes
.venv/bin/python judge_eval.py analyze       # metrics + bootstrap CIs -> results.json
```

`judge_eval.py selftest` unit-tests the mirror/flip/primacy/majority logic in isolation.

## Files

| file | what |
|---|---|
| `judge_eval.py` | the whole pipeline (prep / judge / genchk / analyze / selftest / mkpublic) |
| `run_ladder.sh` | runs judge+genchk for the 4 sizes |

| `pairs_meta.json` | pair-set provenance (counts, filters, seed, template hash) |
| `verdicts_<model>.csv` | RAW verdicts: pair, order, verdict, logprobs of A/B/C, latency |

Answer texts are not committed (they are one `prep` away from the CC-BY-4.0 source
dataset); everything needed to *audit the analysis* is.

## Verbosity-bias extension (problem 4f67decd, finding extends cf4e6c02)

125-pair subset where the human majority preferred the SHORTER answer (63) or tied (62);
three prompt arms (noinstr / default=cf4e6c02 template / antiverb), both orders, same
deterministic verdicts. `verbosity_eval.py`; raw verdicts `verdicts_vb_*.csv`; metrics
`results_vb.json`; subset (no texts) `subset_vb_public.parquet`.

| judge | P(longer) noinstr [CI] | default | antiverb | antiverb−noinstr [CI] |
|---|---|---|---|---|
| 0.5B | 0.476 [0.452, 0.496] | 0.476 | 0.464 | -0.012 [-0.032, 0.004] |
| 1.5B | 0.616 [0.552, 0.676] | 0.588 | 0.596 | -0.020 [-0.048, 0.008] |
| 3B | 0.428 [0.360, 0.496] | 0.412 | 0.344 | -0.084 [-0.128, -0.040] |
| 7B | 0.492 [0.412, 0.572] | 0.472 | 0.472 | -0.020 [-0.056, 0.016] |

Bias is clearly measurable only at 1.5B; the standard anti-length sentence does nothing
significant at any scale; the strengthened instruction acts only at 3B (overcorrection where
no bias existed) and fails at 1.5B where the bias lives. 0.5B is unmeasurable (position-
dominated; see the main finding).
