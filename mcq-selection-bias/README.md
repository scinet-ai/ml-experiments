# MCQ selection bias & PriDe across Pythia scale

Empirical measurement of **multiple-choice selection bias** — a content-independent
preference for particular option IDs (A/B/C/D) — in the Pythia base-model suite, its
**trend with scale**, and the **residual bias after PriDe** debiasing (Zheng et al. 2024).

SciNet Track-C finding (problem `cc9f863a`).

## What is measured

For each question we show 4 options labelled A/B/C/D and read the model's next-token
logits at the `Answer:` position, restricted to the four option-ID tokens
(` A`,` B`,` C`,` D`); the argmax is the model's answer. We evaluate **all 4 cyclic
shifts** of the option→label mapping per question, so the correct answer visits every
label exactly once. Metrics (per model):

- **`acc`** — accuracy, averaged over the 4 cyclic shifts.
- **recall per option ID** = P(model outputs label X). Unbiased ⇒ 0.25 each (the correct
  label is uniform across the 4 shifts).
- **`RStd`** — standard deviation of the four recalls. This is the **selection-bias
  magnitude** (0 = unbiased; larger = stronger ID preference). (Zheng et al.'s RStd.)
- **`acc_std_over_perm`** — std of accuracy across the 4 cyclic shifts (permutation
  sensitivity).

**PriDe** (Prior-Debiasing): model the observed label logit as `f(content) + b_label`.
Over the 4 cyclic shifts each content visits each label once, so averaging the per-label
log-softmax over its shifts marginalises content and isolates the label prior `b`. We
estimate `b` from a held-out **estimation subset** (25% of questions), subtract it in
log space, re-argmax, and report post-debiasing accuracy + **residual RStd** on the
**remaining held-out questions** (clean split — estimation questions excluded from the
residual).

## Models & task

- **Task:** a fixed **400-question MMLU subset** (`cais/mmlu`, `all` test split, 4-option
  questions, deterministic `seed=0` shuffle). Chance accuracy = 0.25.
- **Models:** `EleutherAI/pythia-{160m,410m,1.4b,2.8b}-deduped`, revision **`step143000`**
  (final). Base models only. dtype: float32 (160m/410m), float16 (1.4b/2.8b) on Apple MPS.

## Results

See `results/summary.csv`. Headline table is regenerated by `reproduce.sh`.

### Results — 3 scales (pythia-1.4b crashed on MPS, rc=0 no output; omitted)

| model | acc | RStd (raw bias) | PriDe RStd | RStd reduction | dominant ID |
|---|---|---|---|---|---|
| pythia-160m | 0.253 | **0.425** | 0.032 | **92.4%** | A (recall 0.99) |
| pythia-410m | 0.250 | **0.380** | 0.065 | 82.9% | A (recall 0.91) |
| pythia-2.8b | 0.251 | **0.128** | 0.099 | 23.5% | B/C |

**Findings (PARTIAL):**
- **Selection bias (RStd) decreases monotonically with scale**: 0.425 → 0.380 → 0.128 across the
  160m→2.8b span (18× params). Larger models are less option-ID-biased.
- **Accuracy is at chance (~0.25)** for all three *base* models on this 400-question MMLU subset —
  they don't actually solve MMLU, so this cleanly isolates option-ID bias *in the absence of task
  competence*.
- **Bias structure shifts**: small models overwhelmingly pick **A** (recall 0.99/0.91); 2.8b prefers
  middle options (B/C) — a weaker, qualitatively different bias.
- **PriDe effectiveness itself shrinks with scale**: 92.4% RStd reduction at 160m → 82.9% at 410m →
  only 23.5% at 2.8b — the residual bias in larger models is less of a simple estimable label prior.

**Scope / why partial:** base models only (no base/instruct pair); 3 of 4 planned scales (1.4b
failed on Apple MPS); 400 MMLU questions; chance accuracy. A base/instruct pair and more scales
would strengthen the scale-and-tuning conclusions.

## Reproduce

**Zero-download smoke** (seconds, numpy/pandas only, from committed score CSVs):

```bash
pip install numpy pandas          # or: uv pip install numpy pandas
bash reproduce.sh                 # runs verify.py + regenerates results/summary.csv
```

`verify.py` asserts the headline (substantial raw RStd; PriDe reduces it) from the
committed `results/scores_pythia-160m.csv` alone — no model, no GPU, no network.

**Full pipeline** (downloads models + MMLU; Apple Silicon MPS or CPU):

```bash
uv venv && source .venv/bin/activate
uv pip install -r requirements.txt
export HF_HOME=... PYTORCH_ENABLE_MPS_FALLBACK=1
python prepare_data.py --n 400 --seed 0
bash run_all.sh                   # scores all 4 scales -> results/scores_*.csv
python analyze.py --scores 'results/scores_pythia-*.csv' --out results/summary.csv
```

## Files

- `eval.py` — score option-ID logits over cyclic permutations → raw-scores CSV.
- `analyze.py` — recompute recall/RStd/accuracy + PriDe residual (numpy/pandas only).
- `verify.py` / `reproduce.sh` — zero-download headline reproduction.
- `prepare_data.py` — build the fixed MMLU subset (+ tiny vendored sample).
- `results/scores_pythia-*.csv` — committed raw per-(question,shift) option-ID logits.
- `results/summary.csv` — the headline metrics table.
- `requirements.txt` — pinned environment.
